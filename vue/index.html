vue项目性能优化实战总结

1. 分析项目bundle大小​

report

- 使用webpack-bundle-analyzer来生成一个报告，可以清晰直观的看出打包后文件的大小
-我们在vue项目的package.json文件中加入下面的命令
"build-report": "vue-cli-service build --report"
然后，在命令行中运行
npm run build-report
等一段时间后，会在dist文件中生成一个report.html文件，我们在浏览器中打开它

build

    - 在本地使用打包命令
    - npm run build
    - 然后再终端输出信息

可以看到主要的文件从大到小的排序

2. 懒加载模块​

比如Echars图标的包和编译器的包，都是很大的45百kb，但是不是所有的页面都需要图表和编辑器，只有少量页面需要，所以我们可以把Echars和编辑器放到需要的页面去加载，不要放到全局里面，这样其他页面加就省去他们的时间
要查看浏览器加载文件的顺序，我们打开浏览器的开发者工具，点击Network，勾选上Disable cache，这样不会从浏览器的缓存中去加载文件，然后重新去刷新页面，可以看到所有文件的加载顺序 一般我们引入模块是这样写的

// demo.js
const Demo = {
  testDemo: function () {
    console.log("这是DEMO")
  }
}
export default Demo

// app.js
import Demo from './demo.js'
Demo.testDemo()

使用懒加载这样写

// app.js
const getDemo = () => import('./demo.js')

// 延迟加载
getDemo()
  .then({ testDemo } => testDemo())

懒加载能过非常好的解决bundle过大的问题，把一些用户并不常用的功能放到懒加载中，放用户点击或者滚动可视区在进行加载

3. 路由懒加载​

一般我们的路由可能时这样写，这样写在打包时把Dashboard组件和Contact组件的代码全部打包到js中去中

// routing.js
import Dashboard from './Dashboard.vue'
import Contact from './Contact.vue'

const routes = [
  { path: '/', component: Dashboard }
  { path: '/contact', component: Contact }
]

我们可以按路由懒加载，改成下面这样

// routing.js 
const routes = [
  { path: '/', component: () => import('./Dashboard.vue') }
  { path: '/contact', component: () => import('./Contact.vue') }
]

这样的好处是，我们没有访问路由时/contact，就不会去加载Contact.vue的代码，只有访问时才会去加载组件的代码

通过webpackChunkName来聚合打包多个页面到一个bundle中 一般我们的一个页面的bundle可能非常小，我们可以把一个模块的所有路由的页面都打包到一个bundle中，在import函数中加入/ webpackChunkName:'group-superAdmin' /webpackChunkName的名字可以自己定义

const router = [
  {
    path: 'superAdminAccountList',
    name: 'SuperAdminAccountList',
    component: () => import(/* webpackChunkName:'group-superAdmin' */ '@/activity/superAdmin/AccountList'),
  },
  {
    path: 'superAdminCreateAccount',
    name: 'SuperAdminCreateAccount',
    component: () => import(/* webpackChunkName:'group-superAdmin' */ '@/activity/superAdmin/CreateAccount'),
  },
  {
    path: 'superAdminRoleList',
    name: 'SuperAdminRoleList',
    component: () => import(/* webpackChunkName:'group-superAdmin' */ '@/activity/superAdmin/RoleList'),
  },
  {
    path: 'superAdminCreateRole/:id?/:look?',
    name: 'SuperAdminCreateRole',
    component: () => import(/* webpackChunkName:'group-superAdmin' */ '@/activity/superAdmin/CreateRole'),
  },
];

4. prefetch预渲染组件​

- prefetch是一种利用浏览器的空闲时间加载将来可能用到的资源的一种机制；通常可以利用于加载非首页的其他页面所需的资源，以便加快后续页面的首屏速度

通过webpack我们可以非常方便的实现组件的预渲染，通过加入/* webpackPrefetch: true */

components: {
  ModalView: () => import(/* webpackPrefetch: true */ './ModalView.vue')
}

6. 善用浏览器缓存​

我们都知道我们访问一个网站后，浏览器会对JS和CSS去进行缓存，在下次访问的时候会从缓存中去获取数据，不用在去请求。可能你会文浏览器缓存我们也控制不了？我们能怎么做? 一般我们打包后的文件像这样

- main.[hash].js – 根组件
- common.[hash].js – 公共组件
- dashboard.[hash].js – dashboard页面
- contact.[hash].js – contact页面

其实我们可以把一些很长时间都不会需要的公共代码放到common[hash].js中去，这样依次缓存后，以后再次访问就不会去发请求了

7. 优化压缩图片​

图片的大小对于整个项目的性能也是至关重要的，一般webpack会帮助我们把小的图片转化位base64来减少网络请求，对其他的图片来说我们也要进行压缩，一般压缩方式有两种

1. 使用软件来进行压缩
2. 使用CDN来进行文件压缩

使用软件压缩我们推荐使用TinyPNG，一个在线网站，使用它来压缩几乎不损失清晰度而且压缩的效果特别好

使用CDN压缩，一般专业的文件存储都会提供图片的处里功能 比如京东的图片地址，中间的s280x280可以去修改图片的宽高，后面的webp后缀.webp后缀就是把jpg图片转换成webp格式，进一步的来缩小文件的大小 https://img30.360buyimg.com/seckillcms/s280x280_jfs/t1/187871/40/5785/69529/60b4af7dE5a50eaff/c235bb87f19c1698.jpg.webp

8. 静态文件CDN​

一般小公司可能为了方便直接把CSS、JS、图片等文件直接上传到服务器上进行访问，使用CDN的优势在于CDN是全国的地方都会有的服务节点，而且CND也会缓存文件，所以通过CND访问静态文件和直接访问服务器文件要块上几倍 http://www.example.com/images/logo.png
使用CDN后的地址为：
http://cdn.example.com/images/logo.png
